核心原则:

  • 保持简单: 从最简单的方案开始，只在必要时增加复杂性。80% 的任务可以用单 Agent + 工具调用完成
  • 透明性: 明确展示 Agent 的规划步骤，让用户了解决策过程
  • 渐进式自主: 从低自主度开始，逐步建立信任后再增加权限

  • 单 Agent 模式: 适合简单任务，响应快，易调试
  • 多 Agent 顺序模式: 适合多阶段任务，每个 Agent 负责一个阶段
  • 多 Agent 并行模式: 适合需要并行处理的任务，需要协调者
  • 层级管理模式: 适合复杂任务，有明确的上下级关系
